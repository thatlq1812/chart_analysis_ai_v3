\section{Results and Discussion}

This section presents experimental results from each pipeline stage and the end-to-end
system, followed by analysis, comparison with related work, and recommendations.

\subsubsection{Stage 2: Chart Detection (YOLOv8m)}

The YOLOv8m detector was trained on the chart detection dataset (approximately 32,440
images, single-class ``chart'') for 100 epochs with SGD optimizer (lr=0.01, momentum=0.937).
\Cref{tab:yolo} presents the detection results.

\begin{table}[htbp]
\centering
\caption{YOLOv8m chart detection results.}
\label{tab:yolo}
\input{figures/tables/tab_yolo_results}
\end{table}

The detector achieves 93.5\% mAP@0.5, exceeding the target of 85\%. The single-class
strategy simplifies training and avoids category confusion at the detection stage,
delegating type classification to a dedicated model.

Grad-CAM \cite{selvaraju2017gradcam} visualizations confirm that ResNet-18 focuses on

for pie charts. This validates that the model learns meaningful chart structure rather
than background artifacts.

\subsection{Discussion}
% ============================================================

\subsubsection{Hybrid Architecture Validates Core Hypothesis}

The experimental results support the core hypothesis that geometric precision combined
with AI reasoning outperforms pure end-to-end approaches for chart data extraction:

\begin{itemize}
    \item 100\% extraction success rate (Stage~3) demonstrates that geometric methods are
          robust across 8 chart types and 32,364 diverse academic charts.
    \item 93.5\% detection mAP and 94.14\% classification accuracy confirm that the
          neural perception components are effective.
    \item 92.6\% overall pipeline confidence before SLM integration indicates that the
          geometric foundation alone provides a strong baseline.
\end{itemize}

By separating perception (neural) from measurement (geometric) from reasoning (SLM),
the system provides \textbf{full traceability}: every extracted value can be traced back
to a pixel coordinate, an OCR text, and a calibration model, unlike end-to-end approaches
where outputs are opaque neural predictions.

\subsubsection{Comparison with Related Work}

\Cref{tab:related_cmp} compares Geo-SLM with existing methods.

\begin{table}[htbp]
\centering
\caption{Comparison with related chart understanding methods.}
\label{tab:related_cmp}

Key differences from end-to-end approaches (DePlot \cite{liu2023deplot},
MatCha \cite{lee2023matcha}):

\begin{itemize}
    \item \textbf{Measurement vs.\ estimation}: Geo-SLM computes data values via
          geometric calibration (RANSAC + linear regression), while DePlot/MatCha
          estimate values from visual patterns. Geometric measurement is deterministic
          and verifiable.
    \item \textbf{Explainability}: Every intermediate result (skeleton, polylines,
          calibration parameters, OCR text) is inspectable. End-to-end models produce
          only final predictions.
    \item \textbf{Offline capability}: The local SLM (1.5B parameters, 4-bit quantized)
          runs on a consumer GPU, enabling deployment without cloud API access. DePlot
          and MatCha require $\geq$282M-parameter models.
    \item \textbf{Chart type coverage}: 8 types across a diverse academic corpus,
          compared to 3--5 types for most existing systems.
\end{itemize}

\subsubsection{Modularity Enables Iterative Improvement}

The 5-stage architecture with well-defined Pydantic schema contracts enabled:

\begin{itemize}
    \item Replacing PaddleOCR with EasyOCR without modifying any pipeline interface.
    \item Adding ResNet-18 classification without changing the Stage~3 output schema.
    \item Planning SLM integration (Stage~4) without touching geometric extraction code.
    \item A 9.9$\times$ improvement in training data quality (v2 $\to$ v3) by fixing
          upstream extraction errors isolated to the geometric mapper submodule.
\end{itemize}

\subsubsection{Core-First Strategy Validates Vietnamese Readiness}

The decision to build and validate the pipeline with English data before localizing to
Vietnamese (see \Cref{sec:viet_loc}) is empirically justified by the results:

\begin{enumerate}
    \item \textbf{Isolated debugging}: The 100\% extraction success rate (Stage~3) and
          92.6\% pipeline confidence were achieved without any language-specific code.
          This confirms that the geometric core is stable and that any future Vietnamese
          errors can be attributed to the localization layer, not the pipeline itself.
    \item \textbf{Language dependency audit}: Analysis of the 17 major pipeline components
          (\Cref{tab:lang_analysis}) reveals that 12 are fully language-agnostic
          (detection, classification, skeleton extraction, vectorization, calibration,
          element detection). Only 4 require additive changes (keyword dictionaries,
          prompt templates) and 1 requires a configuration change (OCR language).
    \item \textbf{Adapter pattern validation}: The AI routing layer processes Vietnamese
          and English identically -- adapters pass prompt strings to models without
          language-specific logic. Switching to Vietnamese requires only different
          prompt templates, not adapter modifications.
    \item \textbf{OCR engine flexibility}: The existing \texttt{OCRConfig.languages}
          parameter supports multi-language lists. The switch from \texttt{["en"]} to
          \texttt{["vi"]} or \texttt{["en","vi"]} is a single-line configuration
          change with no code modification.
\end{enumerate}

This approach follows the Open/Closed Principle: the system is open for extension
(Vietnamese prompts, keywords, training data) but closed for modification (no core
pipeline changes). Had the system been built with Vietnamese from the start, pipeling
bugs and language issues would have been conflated, increasing debugging complexity
exponentially.

\subsubsection{Limitations of Current Results}

\begin{enumerate}
    \item \textbf{No public benchmark comparison}: Geo-SLM has not yet been evaluated
          on ChartQA \cite{masry2022chartqa} or PlotQA \cite{methani2020plotqa}.
          This is planned upon completion of SLM fine-tuning.
    \item \textbf{SLM training in progress}: Stage~4 reasoning results are from Gemini
          API prototyping. Local SLM evaluation will be supplemented.
    \item \textbf{Processing speed}: 7.6~seconds per chart exceeds the target of
          $<$5~seconds. OCR optimization is the primary bottleneck.
    \item \textbf{Domain specificity}: All training data comes from academic papers
          (arXiv). Generalization to business charts and dashboards is untested.
    \item \textbf{Vietnamese evaluation pending}: The Vietnamese localization components
          (OCR configuration, keyword dictionaries, prompt templates, Vietnamese QA
          training data) are designed but not yet evaluated end-to-end. Vietnamese
          OCR accuracy with diacritical marks and SLM Vietnamese output quality are
          pending validation.
\end{enumerate}

\subsection{Recommendations}
% ============================================================

Based on the results and identified limitations, we recommend the following directions
for improvement:

\begin{enumerate}
    \item \textbf{Complete SLM fine-tuning and evaluation}: Train and evaluate the
          Qwen-2.5-1.5B model on the 268,799-sample dataset. Target metrics: JSON valid
          rate $>$95\%, field accuracy $>$90\%, numeric accuracy $>$85\%, inference
          latency $<$2~seconds per chart.
    \item \textbf{Vietnamese QA integration}: Execute the Vietnamese localization plan
          (\Cref{sec:viet_loc}): configure Vietnamese OCR, generate 5,000--10,000
          Vietnamese QA samples, fine-tune SLM with combined English + Vietnamese data,
          and evaluate Vietnamese chart understanding accuracy.
    \item \textbf{Public benchmark evaluation}: Evaluate on ChartQA and PlotQA to enable
          direct comparison with DePlot, MatCha, and other published methods.
    \item \textbf{Ablation study}: Measure the contribution of key components (negative
          image transform, adaptive RDP, RANSAC calibration, SLM reasoning) through
          systematic ablation experiments.
    \item \textbf{OCR optimization}: Implement batch OCR processing and selective
          re-execution to reduce Stage~3 processing time by an estimated 40--50\%.
    \item \textbf{Domain expansion}: Collect and annotate chart images from Vietnamese
          business reports and government dashboards to evaluate cross-domain and
          cross-language generalization.
    \item \textbf{Dataset release}: Prepare the training dataset (English + Vietnamese)
          and model weights for public release to support community research.
\end{enumerate}
