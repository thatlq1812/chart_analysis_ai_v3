### Phase 2: Root Cause Analysis

**Finding: Annotations were empty shells.**

All 50 annotation files in `data/benchmark/annotations/` only had `chart_type` and `difficulty` populated. Critical fields were all empty:

| Field | Value | Impact |
| --- | --- | --- |
| `elements.element_count` | 0 | Evaluator skips element accuracy |
| `axis.x_min/x_max/y_min/y_max` | null | Evaluator skips axis accuracy |
| `texts` | [] | OCR evaluation impossible |
| `title` | null | Title recall = 0% |
| `data_series` | [] | No value comparison |

The evaluator logic is correct - it guards with `if gt_element_count > 0` before computing element accuracy. With all GTs at 0, the denominator is 0 -> 0%.

**Element Count = 0%**: Mean error = 1598%. Stage 3 massively over-detects:

| line (3) | 2-3 | 4-433 | 100% - 14333% |

Two issues at play:
1. GT counts (from QA) measure **semantic categories** (3 countries, 2 series)
2. Stage 3 counts **individual detected elements** (bars + points + false positives)
3. Stage 3 also has real over-detection (97 "slices" for a pie with 5 slices)

**Axis Range = 0%**: Stage 3 returns `axis_detected: false` for all charts. Axis calibrator is non-functional on these real-world charts.

## Phase 9: Evaluator Improvement

Modified `scripts/evaluation/benchmark/evaluate.py` to use **type-filtered element counting**:

Before: `pred_count = len(pred_elements)` (counts ALL detected elements regardless of type)
After: `pred_count = sum(1 for e in pred_elements if e.type == gt_type)` (only counts matching types)

Rationale: Stage 3 detects mixed element types for every chart (bars+points+slices even for scatter charts). Counting all types inflates the error. Wrong-type detections are already captured by the separate `type_correct` metric.

### Axis Detection: Completely Non-Functional

Stage 3 returns `axis_detected: false` and all axis values as null for **every single chart** (67/67 axis values = null). The axis calibrator module is either:
1. Not triggered correctly by the pipeline
2. Internal thresholds too strict
3. Geometry detection failing on real-world chart images

### Root Causes of Stage 3 Weaknesses

1. **Over-detection**: Stage 3 detects ALL element types (bar+point+slice) regardless of chart type. A scatter chart gets bars and slices detected alongside points.
2. **Element counting**: Even type-filtered, S3 detects far more elements than actually exist (e.g., 8 slices for a 1-slice pie chart, 33 points for a 5-line chart).
3. **Axis calibrator broken**: Returns all nulls. Axis detection thresholds may be calibrated for synthetic charts, not real-world academic images.
4. **OCR returns nothing**: `texts: []` for all 50 charts. OCR pipeline produces no output.
